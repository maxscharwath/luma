# Rebrand LUMA → Kroma

Full rename, not just the display name. Scope from a repo scan: **~618 files**, ~3260 `luma`
occurrences, 39 Rust crates, 18 npm packages, 12 modules, 44 `LUMA_*` env vars, 132 `.lmod` refs.

Domain: **kroma.tv**. Reverse-DNS base becomes **`tv.kroma`** (matches the domain).

## Identifier mapping (the contract)

| Kind | Before | After |
|---|---|---|
| Display / brand | `LUMA` / `Luma` | `Kroma` |
| npm workspace scope | `@luma/*` | `@kroma/*` |
| Rust crates + dirs | `luma-*` (39) | `kroma-*` |
| Server binary | `luma-server` | `kroma-server` |
| Module ids + dirs | `dev.luma.*` (12) | `tv.kroma.*` |
| Module bundle ext | `.lmod` | `.kmod` |
| Env vars | `LUMA_*` (44) | `KROMA_*` |
| DSM package id | `luma` | `kroma` |
| DSM install path | `/var/packages/luma` | `/var/packages/kroma` |
| Docker image | `ghcr.io/maxscharwath/luma` | `ghcr.io/maxscharwath/kroma` |
| webOS app id | `app.luma.webos` | `tv.kroma.webos` |
| Android appId | `app.luma.tv` | `tv.kroma.androidtv` |
| Tauri identifier | `app.luma.desktop` | `tv.kroma.desktop` |
| Tizen widget | (in config.xml) | `tv.kroma.tizen` |
| GitHub repo | `maxscharwath/luma` | `maxscharwath/kroma` (your action) |
| Package source URL | `maxscharwath.github.io/luma` | `packages.kroma.tv` (worker) |

Env-var casing: `LUMA_` → `KROMA_` (same UPPER_SNAKE). Nested config prefixes seen
(`LUMA_API__`, `LUMA_MPV__`, `LUMA_ANDROID__`) follow the same rule.

## ⚠️ Irreversible / breaking (accepted for a pre-1.0 rebrand)

1. **DSM in-place upgrade breaks.** Changing the package id `luma`→`kroma` means an
   existing NAS sees Kroma as a *different* package: it installs fresh at
   `/var/packages/kroma`, does **not** carry over `/var/packages/luma/var` (DB, config,
   image cache, Whisper model). Only affects machines that already run LUMA. Options:
   (a) accept fresh install + document a manual `var/` copy, or (b) ship a one-shot
   migration in `postinst` that copies the old data dir if present. Recommend (b).
2. **Installed modules re-key.** `dev.luma.*` → `tv.kroma.*` means any already-installed
   `.lmod` won't match the new ids; they'll re-install as `.kmod`. Fine pre-1.0.
3. **Docker tag change** `ghcr.io/.../luma` → `.../kroma`: existing `docker pull` refs and
   any compose files must update. Old tags stay pullable; new pushes go to the new path.
4. **Desktop auto-update**: the Tauri `identifier` change makes updaters treat it as a new
   app (one-time reinstall for existing desktop users). Pre-1.0, fine.

The `.kmod` bundle format itself is unchanged (magic = zstd/gzip/tar, no "LUMA" bytes), so
old and new bundles are structurally identical — only the extension string moves.

## Execution phases (each ends with a green build gate)

### Phase 0 — Prep
- Branch `rebrand/kroma`.
- Snapshot: `cargo build --workspace` + `bun run typecheck` + `bun run test` all green FIRST
  (so any post-rename breakage is attributable).

### Phase 1 — Structural: Rust workspace (highest risk, do first)
- Rename 39 crate dirs `server/crates/luma-*` → `kroma-*` and 12 `server/modules/dev.luma.*`
  → `tv.kroma.*` via `git mv` (preserves history).
- Rewrite every `Cargo.toml` `name = "luma-*"`, all `[dependencies] luma-* =`, all
  `path = "../luma-*"`, all `use luma_*::` / `luma_*::` (underscore form!), `extern crate`.
- `luma-server` → `kroma-server` (binary name in Cargo.toml + `[[bin]]` + Dockerfiles).
- `include_str!` paths that reach into `packages/core/src/locales` are unaffected (path, not id).
- Gate: `cargo build --workspace` green.

### Phase 2 — Structural: module system
- `module.json`/`.module.md` ids `dev.luma.*` → `tv.kroma.*` (12 modules + the reserved/test
  ids `dev.luma.a`, `dev.luma.lib`, `dev.luma.reserved`, `dev.luma.hellowasm`, `dev.luma.notes`,
  `dev.luma.engine.example`).
- `.lmod` → `.kmod` everywhere: `scripts/pack-module.ts`, `module-format.ts`, `gen-registry.ts`,
  `lmod-build-plan.ts` (+ rename the file to `kmod-build-plan.ts`), `LMOD_*` env
  (`LMOD_TARGET`/`LMOD_SKIP_BUILD` → `KMOD_*`), kernel/supervisor/manifest `.rs`, web
  `admin/modules.tsx`, all module `Cargo.toml` comments.
- Regenerate: `bun run modules:gen` (rewrites `luma-modules-generated`/`modules-generated`).
- Gate: `bun run modules:validate` + `cargo build --workspace`.

### Phase 3 — Structural: npm workspaces
- 18 `package.json` names `@luma/*` → `@kroma/*` + every import specifier
  (`@luma/ui` → `@kroma/ui`, …), `tsconfig` path aliases, vite aliases, the `#web`/`#kroma`
  internal aliases if any, `bun.lock` (regenerated by `bun install`).
- Rename `packages/ui/src/components/LumaIntro` → `KromaIntro` (+ its imports).
- Gate: `bun install` + `bun run typecheck`.

### Phase 4 — Env vars + config
- `LUMA_*` → `KROMA_*` (44 vars) across server config loader, all client build scripts,
  CI workflows, `.env.example`, docker/compose, desktop sidecar env, README/INSTALL.
- `LUMA_MAGIC` (host-token magic) → `KROMA_MAGIC`.
- Gate: server boots locally reading `KROMA_*`; `bun run dev` works.

### Phase 5 — Client app identifiers + packaging
- webOS `appinfo.json` id/vendor/title; Tizen `config.xml` widget id + name (public + dist +
  .dev-shell); Android `applicationId`/`namespace`/manifest `package` + the Kotlin package dir
  `app/luma` → `tv/kroma` + `assets/luma` asset dir + ExoBridge refs; Tauri `identifier` +
  productName + window titles.
- DSM: `INFO.template` `package="luma"`→`"kroma"` + displayname; `spk/scripts/*`
  (`PKG="luma"`, `/var/packages/luma/...`), optional data-migration in `postinst`.
- Gate: each client build (`bun run build:web|tizen|webos|androidtv`) green; `.spk` builds.

### Phase 6 — CI/CD + infra
- Workflows: image name, artifact names (`luma-*` → `kroma-*`), `.spk` filename
  (`luma-<v>-x86_64.spk` → `kroma-…`), nightly asset `luma-nightly-x86_64.spk`,
  `.info.json` sidecar, the package-source worker (`GITHUB_REPO`, `luma-packages` →
  `kroma-packages`, icon path, landing copy), Synology README URLs.
- `packages.kroma.tv` custom domain on the worker.
- GitHub repo rename `luma`→`kroma` (your action; GitHub keeps redirects). Update the 16
  `maxscharwath/luma` refs.

### Phase 7 — Brand strings, UI, docs, assets
- All visible `LUMA`/`Luma` → `Kroma`: `@kroma/ui` Logo/Intro, TV + web UI copy, i18n catalogs
  (`packages/core/src/locales/*.json`, fr + en), page `<title>`s, meta, PWA manifest name,
  admin console labels, tagline → **"Color, exactly as filmed."** (pending your pick).
- Docs: `README.md`, `ARCHITECTURE.md`, `INSTALL.md`, `CONTRIBUTING.md`, `docs/*`,
  `modules/README.md`, `clients/**/README.md`.
- Logo/icon: the current gold-ring mark; decide keep vs. new Kroma mark (separate design task).
- Update the memory files' brand references (`.claude/.../memory/*`) as a follow-up.

### Phase 8 — Full verification
- `cargo build --workspace` + `cargo test --workspace` + `bun run typecheck` + `bun run test`
  + every client build + `.spk` build + Docker build-from-spk.
- Grep guard: `grep -rIi luma` (minus vendored/generated) returns only intentional leftovers
  (e.g. a CHANGELOG history note), ideally zero.
- Run the app (`bun run dev`), click through web + one TV shell, install a `.kmod`.

## Bulk-edit safety (from repo memory)
- Exclude from every recursive sed: `node_modules`, `target`, `dist`, `**/generated/**`,
  `.git`, and especially **`clients/synology/.cache`** (gitignored vendored cargo cache that
  naive walkers corrupt). Generated files (`*routeTree.gen.ts`, `luma-modules-generated`,
  `modules-generated`) are regenerated, not hand-edited.
- No em dashes introduced (repo policy).
- Case-preserving replace: `LUMA`→`KROMA`, `Luma`→`Kroma`, `luma`→`kroma`, `luma_`→`kroma_`
  (Rust), run as three separate passes so casing stays correct.
